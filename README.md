# Telegram Lead Bot Template: Полная инструкция для запуска под клиента

Этот README написан для одного сценария:
вы берете этот шаблон, настраиваете под конкретный бизнес, запускаете, и бот стабильно принимает лиды.

Ниже все шаги в правильном порядке. Если идти по ним последовательно, проект должен заработать без сюрпризов.

## 1. Что делает бот

- Принимает заявки от пользователей в Telegram.
- Сохраняет лиды в PostgreSQL.
- Отправляет уведомления менеджеру (админу) в Telegram.
- Поддерживает команды админа:
  - `/stats`
  - `/leads`
  - `/leads_new`
- Имеет напоминания по "зависшим" лидам (сохраняются в БД, не теряются после перезапуска).
- Имеет защиту от дублей webhook-событий по `update_id`.

## 2. Что подготовить перед стартом

На каждого клиента подготовьте:

1. Отдельного Telegram-бота (через BotFather).
2. Отдельный домен или поддомен с HTTPS (например `https://acme-bot.yourdomain.com`).
3. Отдельный сервер/VPS (рекомендуется) или изолированную среду.
4. Telegram ID менеджера клиента (кто принимает лиды).

Важно:
- Этот шаблон рассчитан на модель "1 бот = 1 клиент".
- Для каждого клиента используйте отдельный `.env` и отдельную БД/контейнеры.

## 3. Минимальные требования

- Docker
- Docker Compose
- Домен с рабочим HTTPS до вашего приложения

Опционально для локальной проверки:
- Python 3.11+
- `pip`

## 4. Структура важных файлов

- `app/main.py`: FastAPI webhook endpoint, защита от дублей update_id.
- `app/handlers/start.py`: пользовательский сценарий заполнения лида.
- `app/handlers/admin.py`: команды менеджера и напоминания.
- `app/content.py`: тексты кнопок/контактов/сообщений (это меняется под клиента).
- `docker-compose.yml`: запуск БД + бота.
- `.env.example`: пример переменных окружения.
- `alembic/versions/*`: миграции схемы БД.

## 5. Первичная настройка под нового клиента

### Шаг 5.1: создайте `.env`

Скопируйте шаблон:

```powershell
Copy-Item .env.example .env
```

Заполните `.env`:

- `BOT_TOKEN`:
  - токен из BotFather (конкретного клиента).
- `ADMIN_ID`:
  - numeric Telegram ID менеджера клиента.
- `HOST_URL`:
  - публичный HTTPS URL без слеша в конце.
  - пример: `https://acme-bot.yourdomain.com`
- `WEBHOOK_SECRET`:
  - длинный случайный секрет (минимум 32 символа).
- `DATABASE_URL`:
  - обычно: `postgresql+asyncpg://postgres:postgres@db:5432/postgres`

### Шаг 5.2: адаптируйте тексты под клиента

Отредактируйте `app/content.py`:

- контакты (`CONTACTS_TEXT`)
- отзывы (`REVIEWS_TEXT`)
- формулировки полей

Рекомендация:
- Не оставляйте заглушки типа `hello@example.com` и `@your_manager`.

### Шаг 5.3: убедитесь, что webhook URL реальный

Webhook строится как:

`HOST_URL + /telegram/webhook`

Проверьте заранее, что этот URL будет доступен извне.

## 6. Запуск через Docker Compose (рекомендуется)

### Шаг 6.1: старт контейнеров

```powershell
docker compose up -d --build
```

### Шаг 6.2: проверьте статус

```powershell
docker compose ps
```

Ожидаемо:
- `db` healthy
- `bot` running

### Шаг 6.3: проверьте health endpoint

```powershell
curl http://localhost:8000/health
```

Ожидаемо:

```json
{"status":"ok"}
```

### Шаг 6.4: проверьте логи при первом старте

```powershell
docker compose logs bot --tail=200
docker compose logs db --tail=200
```

Проверьте, что:
- миграции применились без ошибок;
- webhook установлен;
- нет traceback.

## 7. Обязательная проверка после запуска (чеклист)

### Чеклист A: пользовательский путь

1. Откройте бот клиентом в Telegram.
2. Нажмите `/start`.
3. Пройдите путь создания заявки:
   - имя
   - телефон
   - услуга (опционально)
   - комментарий (опционально)
4. Убедитесь, что пользователь получил подтверждение.

### Чеклист B: менеджерский путь

1. Проверьте, что менеджеру пришло уведомление о новом лиде.
2. Нажмите `Done` в карточке лида.
3. Выполните команды:
   - `/stats`
   - `/leads`
   - `/leads_new`
4. Убедитесь, что статус лида изменился на `completed`.

### Чеклист C: webhook и дубли

1. Убедитесь, что один и тот же update не создает дубль заявки.
2. При временных сетевых проблемах не должно появляться 2 одинаковых лида от одного update.

## 8. Тесты перед сдачей клиенту

Запуск:

```powershell
pytest -q
```

Перед сдачей клиенту тесты должны быть зелеными.

## 9. Как обновлять бота у существующего клиента

Порядок обновления:

1. Сделайте бэкап БД.
2. Обновите код.
3. Пересоберите контейнеры:

```powershell
docker compose up -d --build
```

4. Проверьте:
   - `docker compose ps`
   - `/health`
   - тестовая заявка через Telegram

Если после релиза что-то не так:
1. Смотрите `docker compose logs bot --tail=300`.
2. Проверяйте переменные в `.env`.
3. Проверяйте доступность `HOST_URL`.

## 10. Резервные копии (минимум, который нужен)

Рекомендуется:
- ежедневный backup БД;
- хранение хотя бы 7 последних копий.

Пример команды (внутри окружения с доступом к `pg_dump`):

```powershell
pg_dump -h <db-host> -U <db-user> -d <db-name> > backup.sql
```

## 11. Безопасность (короткий обязательный минимум)

1. Не коммитьте `.env` в git.
2. Используйте длинный `WEBHOOK_SECRET`.
3. Не переиспользуйте `BOT_TOKEN` между клиентами.
4. Не открывайте PostgreSQL наружу без причины.
5. Используйте HTTPS для `HOST_URL`.

## 12. Типовой процесс "новый клиент за 30-60 минут"

1. Создать бота через BotFather.
2. Получить Telegram ID менеджера.
3. Настроить поддомен + HTTPS.
4. Скопировать `.env.example` -> `.env` и заполнить.
5. Обновить `app/content.py` под клиента.
6. `docker compose up -d --build`.
7. Пройти чеклисты A/B/C.
8. Передать клиенту и наблюдать первые лиды.

## 13. Частые проблемы и решение

Проблема: менеджеру не приходят лиды.
- Проверьте `ADMIN_ID` (должен быть числом и корректным Telegram ID).
- Проверьте, что менеджер хотя бы раз запускал бота.
- Проверьте логи `bot`.

Проблема: webhook не работает.
- Проверьте, что `HOST_URL` публичный и по HTTPS.
- Проверьте, что путь `/telegram/webhook` доступен извне.
- Проверьте `WEBHOOK_SECRET` в `.env`.

Проблема: бот стартует, но падает.
- Обычно это неверные переменные `.env` или недоступная БД.
- Проверяйте traceback в логах контейнера.

## 14. Что менять под каждого клиента (короткая памятка)

Обязательно меняете:
- `BOT_TOKEN`
- `ADMIN_ID`
- `HOST_URL`
- `WEBHOOK_SECRET`
- тексты/контакты/отзывы в `app/content.py`

Рекомендуется менять:
- имя контейнеров/проект Docker Compose (если держите много клиентов на одном хосте).

## 15. Финальная проверка перед продажей

Перед тем как взять деньги с клиента, подтвердите:

1. Бот принимает и сохраняет лиды.
2. Менеджер получает лиды и может закрыть `Done`.
3. Напоминания приходят по зависшим лидам.
4. Нет дублей по одному update Telegram.
5. Есть рабочий backup-процесс.
6. Есть ваш внутренний чеклист обновления.
